Environment variables are global variables which can be accessed throught your application.

## T3 Env

This package provides a simple way to define environment variables validation for your app.

### Installation

<Tabs items={["npm", "pnpm"]}>
    <Tab value='npm'>

```bash
npm install @t3-oss/env-nextjs zod
```

    </Tab>
    <Tab value="pnpm">

```bash
pnpm add @t3-oss/env-nextjs zod
```

    </Tab>

</Tabs>

### Create your schema

```typescript title="src/env.ts"
import { createEnv } from "@t3-oss/env-nextjs";
import { z } from "zod";

export const env = createEnv({
  server: {
    DATABASE_URL: z.string().url(),
  },
  client: {
    NEXT_PUBLIC_APP_URL: z.string().min(1),
  },

  runtimeEnv: {
    DATABASE_URL: process.env.DATABASE_URL,
    NEXT_PUBLIC_APP_URL: process.env.NEXT_PUBLIC_APP_URL,
  },
});
```

### Usage

```typescript
import { env } from "@/env";

const db = env.DATABASE_URL;
```
